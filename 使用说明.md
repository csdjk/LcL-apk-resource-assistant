# APK Resource Assistant v5.0.0 使用说明

## 三个独立步骤

1. **下载 APK**：输入商店链接或包名，选择 APKPure、Google Play 匿名或个人账号。游戏分析建议保留全部 Split APK。
2. **解压与分析**：选择单 APK、多个 Split APK、含 APK 的目录或旧任务。程序安全解压每个分包并识别 Unity、Godot、Unreal。
3. **引擎恢复**：选择解压目录或旧任务；程序按引擎调用对应工具。每一步完成后由用户决定是否继续。

工作区与下方结果卡片之间有一条可拖动的水平分隔条。上下拖动即可调整两块区域的高度；下载、解压分析、引擎恢复三个页签会分别记住自己的布局。窗口宽度变化时，各区域会自动横向伸缩。

分析完成后，结果卡片会分别显示引擎版本、脚本运行时、恢复准备状态和可信度。“版本依据”用于查看版本来自原生库、资源头、PCK 或恢复日志；存在冲突时会保留所有结果。使用“复制版本”和“版本下载页”可准备对应编辑器。

右上角“最近任务”最多保存 30 条任务。可搜索、按引擎筛选并继续到解压或恢复阶段；从列表移除只删除本机索引，不会删除 APK 和恢复目录。

## Unity

在“设置”中选择 `AssetRipper.exe`。识别到 Unity 后，程序启动本机随机端口的 AssetRipper，并通过 `/LoadFolder` 自动载入整个 `Extracted`。接口版本不匹配时会复制路径并打开目录，供手动选择。旧任务中的 `AssetRipper_Input` 仍可直接使用。

## Godot

识别到 `.pck`、`.godot`、`.scn/.res` 或项目标记后，程序首次使用会下载并校验 GDRETools。恢复结果写入任务目录的 `Godot_Recovered`。程序会区分运行时、资源与 GDScript 字节码版本；GDScript 推荐标准版 Godot，C# 项目推荐 .NET 版。如果 PCK 加密，在“临时密钥”中输入 64 位十六进制密钥；操作结束即清空，不保存到磁盘。

## Unreal Engine

程序识别：

- UE4/UE5 PAK：由 repak 检查和解包；
- UE5 IoStore：由 retoc 验证 UTOC/UCAS 并解包；
- FModel：浏览准备好的 `Unreal_Input/.../Content/Paks`。

首次使用会下载并校验 repak、retoc 和 FModel。加密容器可在“临时密钥”输入 64 位十六进制 AES 密钥。取消操作会终止对应子进程；已完成的输出会保留。

## 工具缓存与输出

引擎工具缓存在：

```text
%LOCALAPPDATA%\GooglePlayApkDownloader\Tools
```

固定版本和 SHA-256 见 `THIRD_PARTY_NOTICES.md`。程序每次复用缓存前会复验可执行文件完整性。原始 APK 始终保留，重复任务和重复恢复使用新目录，不覆盖已有资源。

新任务目录直接命名为 `<包名>_<时间戳>`，不会再额外套一层包名目录；解压内容统一放在 `Extracted`，其中每个 Split APK 仍保持独立子目录，避免同名资源互相覆盖。
